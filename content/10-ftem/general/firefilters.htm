Selecting a Fire

Constrain the list of wildfires by using one or a combination of the following:

  1. Zooming into a specific area of the map.
  2. Pick a wildfire based on year, or agency if you are a monitor for more than one agency.
  3. Use the search box and enter the wildfire name.
  4. If you need to zoom to a fire on the map, click the Zoom To button under the fire name.

    5. Need to de-select your fire? Move the mouse cursor to the Wildfire tab and click the black X to the left of the wildfire name.

Filter or Get More Information from the Fire List

To filter fires by status or to see the full metadata for a fire, view the wildfires tab in full screen view. This is done by clicking the Hide map button in the top center of the screen. Note that you can also filter the columns within the fire in descending or ascending order by single left-clicking the column title.

Using a Filter in Attribute Table

For this example we will search for fire polygons with a point of origin jurisdiction of Nez Perce - Clearwater National Forest:

  1. Click Attribute Table button in the bottom right of the map.


  2. Use your mouse cursor to set the approximate extent of your area, the attribute table automatically populates based upon visible map extent.
  3. Click the layer displayed along the top of the table that corresponds with your search, in this example "Wildfires: Poly (Year 2018)" is selected.

  4. Click Options and select Filter.

  5. Create your filter expression and click OK. In this example we use "poo_jurisdiction is IDNCF", for querying the table for Point Of Origin Jurisdiction of Nez Perce - Clearwater National Forest (IDNCF).
    • Need to look up the jurisdiction code for your area? Download the Jurisdiction Look-up CSV.

    • Start with creating a filter set so you can include a date constraint. This is needed because once you start filtering, the automatic data filter in FTEM is disabled.

      This constrains the table to attributes that match your search results.

  6. To zoom to a record, select it by clicking the gray area to the left of the attribute row and use the Zoom To option.

Viewing Fires using Add Layers

Using the Add Layers widget, you can turn on Active or Inactive fire perimeters from GeoMac. This adds them to the Layer List and make the easier to see on the map. In the example below "Inactive Fire Perimeters from GeoMac" is selected in Add Layers, and is displayed in Layer List. You can see the purple polygon outlined on the map. Note that you may have to turn off the other layers in Layer List to more clearly see the selected polygons.

Add a Missing Fire or Correct Fire Data

Add a Missing Fire

Wildfire data is pulled into the FTEM system nightly from IRWIN and GEOMAC.

The most common reason a wildfire is not visible in FTEM is a lack of Control Date, Containment Date, or Out Date in the system of record.

  1. If this date remains blank, it is not visible in FTEM until 45 days after the start date.
  2. For those familiar with WFDSS who have the proper permissions, they could update those date fields, and the wildfire would be pulled into FTEM the following day.
  3. Alternatively, an FTEM user could contact their FTEM Agency Administrator to let them know the fire is not appearing, and to check for this issue in the system of record.

For more details see the FTEM Wildfire Technical Topic.

Correct Treatment Data

Wildfire data is pulled into the FTEM system nightly from IRWIN and GEOMAC. If fire data is incorrect, please work with your FTEM Agency Administrator to have the data corrected in the appropriate system.
