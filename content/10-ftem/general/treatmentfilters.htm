Adding or Removing Treatments

In FTEM, the treatment list is automatically populated for each wildfire based on buffering rules. You can add treatments to this list by following the below steps:

  1. Click the Add Treatment(s) button at the top of the treatment tab.

  2. In the dialogue box that appears replace the default buffer, in meters, with a wider buffer of your choosing.

  3. Click Find Interactions.

  4. Any treatment interactions detected pop up as a list. Use the check boxes next to each interaction to select the ones you want. When finished, click Add Selected Treatments.

In some cases, there may be quite a lot of interactions listed. The search box at the top of the list, and the page buttons at the bottom (Previous and Next) help you narrow the list to find specific treatments.

 

To remove treatments from the list use the check boxes next to the treatments to select them, then click the Remove Treatments button. This removes the treatments from the interaction list for the fire; it does not delete the treatment data from the system. Removed treatments can be added back in by clicking the Add Treatments button.

The Treatment list for a fire is shared by all agency personnel monitoring the fire, you may need to collaborate with monitors from adjacent units to make sure you do not remove treatments others will need to monitor.

Filter or Get More Information from the Treatment List

To filter your treatment list by interaction status, monitoring events, or treatment category, or to see the full metadata for a treatment, view the treatments tab in full screen view. This is done by clicking the Hide map button in the top center of the screen. Note that you can also filter the columns within the treatment in descending or ascending order by single left-clicking the column title.

Using a Filter in Attribute Table

For this example we will search for USFS Treatment Polygons associated with Fremont-Winema National Forest:

  1. Click the Attribute Table button in the bottom right of the map.


  2. Use your mouse cursor to set the approximate extent of your treatment area, the attribute table automatically populates based upon visible map extent.
  3. Click the layer displayed along the top of the table that corresponds with your search, in this example "USFS Treatments: Poly Interactions" is selected.

  4. Click Options and select Filter.

  5. Create your filter expression and click OK. In this example we use "unit_name IS Fremont-Winema National Forest".
    • You can use the Protection Jurisdiction to filter your fires by individual forest, refuge, district etc.
    • Need to look up the jurisdiction code for your area? Download the Jurisdiction Look-up CSV.

    • Start with creating a filter set so you can include a date constraint. This is needed because once you start filtering, the automatic data filter in FTEM is disabled.

      This constrains the table to attributes that match your search results.

  6. To zoom to a record, select it by clicking the gray area to the left of the attribute row and use the Zoom To option.

Add Missing Treatments or Correct Treatment Data

Add a Missing Treatment

Treatments are pulled into FTEM nightly from FACTS and NFPORS and must have the following attributes in the system of record:

  • For NFPORS data, the treatment must be NFP (National Fire Plan), or non-NFP.
  • For FACTS data:
    • A treatment must be KP 3 or KP 6.
    • Where duplicate SUIDs (Sub Unit ID) exist, the SUID with the most recent completion date (ActualCompletionDate) will be chosen.
    • If completion date (ActualCompletionDate) is also duplicated, treatments will be filtered by the the Treatment Category (treatment_category) in this order: 1-Fire, 2-Mechanical, 3-Other, 4-Preparation for Treatment.
    • If Treatment Category is also duplicated, treatments will be filtered by FACTS Activity Code by choosing the smallest numerical code in the remaining duplicate SUIDS.
    • If FACTS Activity Code is duplicated, no further filtering will be applied and remaining treatments will be brought in to FTEM.

If your treatment meets the above criteria but is still not visible in the FTEM Treatment list:

  1. Try increasing your search buffer for interacting treatments. This is done after you select your wildfire, but using the Add Treatments button on the Treatments tab. See the Search Buffer Topic for a step by step walk-through of how to do this. You can also add a using the Map, which is especially useful if the treatment is outside of the maximum buffer area.
  2. If the above do not resolve this issue, you may need to follow up with your FTEM Agency Administrator to have your treatment entered in the system of record. Individual agencies may upload records at different intervals.

For more details see the FTEM Treatments Technical Topic.

Correct Treatment Data

Treatments are pulled into FTEM nightly from FACTS and NFPORS. If treatment data is incorrect, please work with your FTEM Agency Administrator to have your treatment data corrected in the appropriate system.
