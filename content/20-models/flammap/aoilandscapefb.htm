Generating Fire Behavior Reports for an Area of Interest

This is a workaround designed for user specified fire behavior inputs to produce model outputs and reports for an Area of Interest. Future functionality in IFTDSS will allow users to run reports and model outputs for an Area of Interest within the Modeling Playground without having to go through the Develop Treatment Alternatives. This functionality is planned but specific timing for the release is uncertain at this time.

Currently IFTDSS generates reports for existing fire behavior runs, but those are performed for an entire landscape. To perform a fire behavior run and report just for a specific Area of Interest (AOI), you need to use the Develop Treatment Alternative task. The report you want for just the AOI is included when comparing two landscapes. Since you are not interested in the comparison part and just the report output for the AOI, you are going to “trick” the system by creating a phony landscape that is identical to your original landscape but has a different name. This will generate a report comparing fire behavior on the two identical landscapes. The information in this report is about fire behavior in the AOI using your own weather inputs.

To run a fire behavior model and generate a report for a specific AOI: 

  1. From the IFTDSS Planning Cycle, go to the Strategic Planning stage, and select Develop Treatment Alternatives.
  2. Clicksplit screen to display a map area to the left and the tabs on the right, then zoom into your AOI.

  3. Select your landscape from the Select Landscape drop-down list, or create one by clickingCreate on Map. If you create a landscape here remember that the landscape extent must be larger than your AOI.

  4. Select your Area of Interest from the drop-down and proceed to the Editing tab.

  5. Select the Edit tab and select your landscape.

  6. Select Add user created edit rule and add the following rule. You are tricking the system to create an edited landscape, but the edit rule you are building won’t change anything:

    • Where elevation is greater than 0, multiply the canopy cover by 1

  7. Click Save, to save your new landscape with a name indicating that you added a rule that won’t make any changes. For example, X1 after the landscape name to indicate you multiplied by 1.

  8. Select the Input tab, add the fire behavior inputs you want to use, and click Save Inputs.

  9. Click the Run tab and initiate the model run for both landscapes by clicking Run Model.

  10. Go to the Compare tab and check the boxes for your “0 landscape” and the edited landscape you created in the previous steps (indicated in this example by X1).

    Click the Compare on map button. When you compare on the map, the first layer displayed is the "Landscape Change" layer. Since the rule you wrote during editing should not effect on the landscape, your "Landscape Change" layer should look similar to the one pictured below, entirely gray which indicates no change. This ensures that you wrote the rule correctly.

  11. To generate summary reports, click Compare in Summary Report thenclick Create Report when prompted. Because we specified an AOI when first beginning the Develop Treatment Alternatives task, the reports generated describe only the area within your AOI. When the report titles appear in blue they are completed are ready to view by clicking on the blue report title.

    You can also access these reports later by locating them in My Workspace, selecting the file, and clicking View Summary .

View and Understand Report Outputs

Now that you have completed all of the steps to generate the reports, you are ready to view and analyze the information needed for your AOI.

Below is the fire behavior report created for the AOI polygon GreatBasinShape. You can see the inputs selected are displayed. The pie chart shows the breakdown of Flame Length within the AOI polygon.

 

You'll notice two identical pie charts here. This is because the Develop Treatment Alternatives task is usually used to compare different landscape edit outcomes. However, in this case we used the tasks to select an AOI and generate a fire behavior report for our landscape, and a “phony” edited landscape, which was needed to complete this task. The duplicate charts and tables may be ignored, or you could download the data from the table within the report and build your own charts and graphs in another application such as Microsoft Excel if you choose.

This report shows the Landscape Layer Summary (Fuel Model) for just the AOI (this is indicated by the name GreatBasinShape which was the name of the polygon used for AOI in this example). Again, there are two sets of data but they are identical and the duplicate may be ignored.
