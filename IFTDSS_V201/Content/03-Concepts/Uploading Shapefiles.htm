Uploading a Shapefile to IFTDSS

From the homepage, click on Data on the menu. Once in the Saved Data Sets screen select the Shapefile Data tab. The Upload Shapefile link is in the upper left corner of the Shapefile Data tab.

Before uploading a shape file ensure you have the 4 needed individual files that compose a shape file data set; .shp .shx .prj & .dbf.

When uploading a shapefile you have 2 options

  1. You may upload the needed files individually
  2. You may upload a zip file with the needed files

You may not have the files in a subdirectory

There should not be more than one shape in a zip folder

Known issue: You will need to import locally-developed shapefiles.

 

Follow the instructions on the “Uploading a Data Set to IFTDSS” page.

If you experience a display issue the problem is most likely a projection issue. Double check your .prj file and try again. If the issue persists you may need to re-projectA project consists of one or more units that are organized and managed as a single endeavor. the file in a GISGeographic information system. A system for capturing, managing, and analyzing geographically referenced data. software package and then try again.

Double check your shape file uploaded properly by clicking on the gear icon in the Action column and click on Edit.

Once the map shows up you will need to zoom in to an appropriate level so you can see the shape. If your shape still isn’t visible ensure the Shapefile Geometry box is checked under the Overlay heading is CHECKED.

If when examining your shape you notice extra classes in the legend for your shapefile geometry, you must have multiple polygons within your shapefile. You can rename and change the color of each of the classes individually. If you wish to change the color of a shape and it has multiple classes you want to be identical in color you will have to copy and paste the color code for each layer.
